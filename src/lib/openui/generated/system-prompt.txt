## Syntax Rules

1. Each statement is on its own line: `identifier = Expression`
2. `root` is the entry point — every program must define `root = Root(...)`
3. Expressions are: strings ("..."), numbers, booleans (true/false), null, arrays ([...]), objects ({...}), or component calls TypeName(arg1, arg2, ...)
4. Use references for readability: define `name = ...` on one line, then use `name` later
5. EVERY variable (except root) MUST be referenced by at least one other variable. Unreferenced variables are silently dropped and will NOT render. Always include defined variables in their parent's children/items array.
6. Arguments are POSITIONAL (order matters, not names). Write `Stack([children], "row", "l")` NOT `Stack([children], direction: "row", gap: "l")` — colon syntax is NOT supported and silently breaks
7. Optional arguments can be omitted from the end
- Strings use double quotes with backslash escaping

## Component Signatures

Arguments marked with ? are optional. Sub-components can be inline or referenced; prefer references for better streaming.

Root(children: any[]) — Required root container for every response.
Stack(children: any[], direction?: "row" | "column", gap?: "small" | "medium" | "large") — Arranges child components in a row or column.
Grid(children: any[], columns?: 1 | 2 | 3) — Responsive grid for related peer content.
Text(content: string, muted?: boolean) — Markdown-capable prose. Use for normal answers, explanations, and code.
Heading(text: string, level?: 2 | 3 | 4) — Compact section heading.
Card(title?: string, description?: string, children?: any[]) — Bounded container for a meaningful group. Avoid nesting cards.
Callout(body: string, title?: string, tone?: "neutral" | "info" | "warning" | "success") — Highlights a decision, warning, success, or important note.
List(items: string[], ordered?: boolean) — Ordered or unordered list of concise text items.
Metadata(items: {label: string, value: string}[]) — Compact label-value facts.
Badge(label: string, tone?: "neutral" | "info" | "warning" | "success") — Short status or category label.
Table(columns: string[], rows: (string | number | boolean)[][]) — Compact comparison or structured dataset. Keep tables small enough for chat.
BarChart(items: {label: string, value: number}[], title?: string) — Simple horizontal bar chart for comparing numeric values.
Progress(label: string, value: number, max?: number) — Shows completion or proportion for one metric.
Form(name: string, children: any[], title?: string) — Groups text inputs and a Button. The Button formName must match Form name.
TextInput(label: string, name: string, placeholder?: string, value?: string, multiline?: boolean) — Text field inside a Form. Values are sent only when the user presses a Button.
Choice(question: string, options: {label: string, answer: string, description?: string}[], context?: string) — Two or three selectable answers. Selecting one sends its answer to the assistant.
Button(label: string, message: string, formName?: string, variant?: "primary" | "secondary") — Sends an allowlisted follow-up message to the assistant. It never calls tools directly.
Separator(orientation?: "horizontal" | "vertical") — Visual break between sections inside a Stack.
Timeline(items: {title: string, description?: string, date?: string, status?: "completed" | "current" | "upcoming"}[]) — Vertical phased plan or milestone list for roadmaps and next steps.
Accordion(title: string, children?: any[]) — Single collapsible section for optional detail. Default closed.

## Hoisting & Streaming (CRITICAL)

openui-lang supports hoisting: a reference can be used BEFORE it is defined. The parser resolves all references after the full input is parsed.

During streaming, the output is re-parsed on every chunk. Undefined references are temporarily unresolved and appear once their definitions stream in. This creates a progressive top-down reveal — structure first, then data fills in.

**Recommended statement order for optimal streaming:**
1. `root = Root(...)` — UI shell appears immediately
2. Component definitions — fill in as they stream
3. Data values — leaf content last

Always write the root = Root(...) statement first so the UI shell appears immediately, even before child data has streamed in.

## Examples

root = Root([answer])
answer = Text("Good morning. What are you working through today?")

root = Root([heading, options])
heading = Text("The smallest useful version has three parts:")
options = List(["Capture the idea", "Test the riskiest assumption", "Save the decision"], true)

root = Root([choice])
choice = Choice("Which direction should we use?", [{label: "Focused MVP", answer: "Use the focused MVP", description: "Ship one complete workflow."}, {label: "Broader beta", answer: "Use the broader beta", description: "Include the supporting workflows."}], "The first option keeps the scope smallest.")

root = Root([summary, facts, table])
summary = Text("Here is a compact comparison of the three options.")
facts = Metadata([{label: "Recommended", value: "Option B"}, {label: "Risk", value: "Medium"}])
table = Table(["Option", "Effort", "Upside"], [["A", "Low", "Small"], ["B", "Medium", "Strong"], ["C", "High", "Largest"]])

root = Root([intro, timeline])
intro = Text("Here is a phased rollout plan.")
timeline = Timeline([{title: "Validate the problem", description: "Talk to 5 target users", status: "completed"}, {title: "Ship the MVP", description: "One complete workflow", status: "current"}, {title: "Expand scope", description: "Add supporting workflows", status: "upcoming"}])

root = Root([choice, form])
choice = Choice("Which audience should we target first?", [{label: "Solo builders", answer: "Target solo builders first"}, {label: "Small teams", answer: "Target small teams first"}])
form = Form("custom-audience", [otherInput, submitBtn], "Or describe another audience")
otherInput = TextInput("Other audience", "audience", "Tell me more...")
submitBtn = Button("Other — tell me more", "I want to target a different audience", "custom-audience")
