## 🟡 Important

### Q2 🟡 Background readiness can overwrite proposal-prefilled dialog fields

**Where**
`src/routes/workspace/+layout.svelte:351`

**Code**
```svelte
if (data.suggestedName?.trim()) promoteName = data.suggestedName.trim();
```

**Why**
`openPromoteDialog` now accepts proposal detail (`src/routes/workspace/+layout.svelte:315`) and immediately starts a background readiness request (`src/routes/workspace/+layout.svelte:330`). When that request returns, it can overwrite the already-prefilled or user-edited project name/summary, which undermines the “proposal appears instantly, refresh in background” behavior.

**Fix**
Track whether the dialog was opened from a proposal and avoid overwriting fields after open; only fill empty fields or update non-editable readiness sections from the background response.

### Q3 🟡 Create remains enabled while readiness loading blocks cancel

**Where**
`src/routes/workspace/+layout.svelte:1707`

**Code**
```svelte
<Button type="submit" disabled={isPromoting || !promoteName.trim()}>
```

**Why**
Cancel is disabled during readiness loading (`src/routes/workspace/+layout.svelte:1702`), but Create project is not. The user can submit while the dialog is in a loading state that otherwise prevents leaving the modal, which is inconsistent and can race with background readiness updates.

**Fix**
Either allow cancel during readiness loading or also disable Create while `isLoadingReadiness` is true; prefer one consistent interaction model.

### S1 🔵 Delimit untrusted chat content in the readiness AI prompt

**Where**
`src/routes/api/workspace/promotion-readiness/+server.ts:96`

**Code**
```ts
prompt: `Review this chat for promotion into a project. Return only valid JSON with this shape: {"suggestedName":"","suggestedSummary":"","strengths":[""],"missingInformation":[""],"keyArtifacts":[""]}. No markdown fences. Keep arrays to 3-5 concise items. Do not invent artifact contents.\n\nThread title: ${thread.title}\n\nRecent transcript:\n${transcript || 'No saved messages yet.'}\n\nLinked artifact metadata:\n${artifacts || 'No linked artifacts.'}`
```

**Why**
Thread title, transcript, and artifact previews are user-controlled and are interpolated directly into the instruction prompt. This endpoint only generates review text, so it is not a critical sink, but delimiters reduce prompt-injection ambiguity and make the prompt easier to reason about.

**Fix**
Wrap transcript/artifact/title sections in explicit untrusted-data delimiters and tell the model to treat those sections as data, not instructions.
